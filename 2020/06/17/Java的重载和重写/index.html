Java的重载和重写

发表于 2020-06-17 | 分类于 Java

重载（overload）

什么是重载呢？

StringBuilder messages = new StringBuilder();
StringBuilder todoList = new StringBuilder("To do:\n")
上面的例子就是重载，也就是说如果有多个方法，这些方法的名字相同、却又不同的参数，这边就产生了重载。

重载需满足的特点
  1. 被重载的方法必须改变参数列表，
  2. 被重载的方法可以改变返回类型，
  3. 被重载的方法可以改变访问修饰符，
  4. 被重载的方法可以声明新的或更广的检查异常，
  5. 方法能够在同一个类中或者在一个子类中被重载。

重写（override）

什么是重写呢？

重写是子类重写父类的方法，返回值类型和参数不能改变。

重写需满足的特点
  1. 参数列表必须完全与被重写方法相同，
  2. 返回类型与被重写类型可以不同，但是必须是父类返回值的派生类，
  3. 访问权限不能比父类中被重写的方法的访问权限低，
  4. 父类的成员方法只能被她的子类重写，
  5. 声明为final的方法不能被重写，
  6. 声明为static的方法不能被重写，但是能够被再次声明，（下面的链接是证明）

    https://blog.csdn.net/yoamy/article/details/51275388

  7. 子类和父类在同一个包中，那么子类可以重写父类的所有方法，除了声明为private和final的方法，
  8. 子类和父类不在同一个包中，那么子类只能重写父类声明为public和protected的非final方法，
  9. 重写的方法能够抛出非强制异常，无论子类能够重写父类是否抛出异常。但是，重写的方法不能抛出新的强制性异常，或者被重写方法声明的更广泛的强制性异常，反之则可以，(这段话没有理解，先放一下)
  10. 构造方法不能被重写，
  11. 如果不能继承一个方法，则不能重写这个方法。
